Global Scope

The framework defines one global variable - app - that is used as a scope for all modules, as well as provides some general methods described below:
app.action - function (target, data)
Main application event loop. This is where all events from application level layout, toolbar and tabs are coming. It is define in /app/core/conf/action.js.
app.ajax ([url], options)
Wrapper for $.ajax() that takes exactly same set of parameters. The main reason for this wrapper is to provide common functionality, such as error handling. If you do all your ajax calls with app.ajax(...) you should not worry about error handling.
app.ajaxError - function (xhr, status, error)
Allows you to define your own error handler for ajax requests. See app/core/conf/utils.js
app.core
Read-Only object that contains information about core settings. For now it has information about loaded modules and config object of core UI elements.
app.error (msg, [title], [callBack])
Common popup for error messages. If popup is already open, it will slide down the error message within the opened popup.
app.get (files, callBack)
Retrieves files from the server and when all files are received calles callBack function passing all files as an array: The frist argument is an array of file paths, the second is a call back function.
app.header (msg)
Sets application header to msg.
app.load (name, [params], [callBack])
Checks if the module name was previously loaded. If it was loaded, it will call module's .render() method. If module was not yet loaded, it will load the module. You can optionally pass an param object that will become part of the module.
For example if you module name is myModule, it will be registered as app.myModule and it will have parameters as app.myModule.params that you can use in modules render() method.
app.include (files)
Includes all provided files in <script> tags. The files parameter is an array of strings.
app.login ()
Function that is called when user needs to login. By default it is empty function defined in app/core/conf/utils.js
app.logout ()
Function that is called when user needs to logout. By default it is empty function defined in app/core/conf/utils.js
app.route - object
An object to work with setting, getting and parsing of routes. See routes
app.start - function ()
Function that will be called when application is initialized and ready. This funciton is defined in /app/core/conf/start.js file and by default loads home module.
